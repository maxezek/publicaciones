Los bloques Scratch de s4a

El entorno de programación estándar de Arduino es Arduino IDE. El lenguaje de programación de alto nivel que usa Arduino IDE es Wiring/Processing que es una variante de Java dirigida a facilitar la programación de este tipo de dispositivos. Las órdenes de escritas de Wiring tienen su equivalencia en un bloque de Scratch en s4a.

Nuestro Arduino tiene una serie de pines numerados que se pueden activar para leer de ellos o para escribir sobre ellos. Por ejemplo, si queremos usar un LED  debemos conectar una patilla a tierra GND (ground es tierra en inglés) y la otra  a un pin de los denominados a digitales, en concreto el 13. El circuito real incluiría más elementos como resistencias para acondicionar el circuito pero básicamente esta es la idea. Como lo que queremos en este ejemplo es usar ese pin para encender el LED debemos entonces preparar ese pin para ello como SALIDA con la orden:

pinMode(13, OUTPUT);

Como era de esperar Wiring usa como base el inglés. OUTPUT/INPUT son dos expresiones predefinidas para indicar que el pin indicado (el 13 en este caso) funcione como SALIDA o ENTRADA cuando corresponda. Fijate que la orden termina con puno y coma, ;.

Para encender el LED debemos incluir en el programa otra orden que activa la salida del pin a ALTO que se corresponde con 5V (¡Ojo! puede usarse una versión con ALTO a 3,3 V). La sentencia a escribir es:

digitalWrite(13, HIGH);

De nuevo la orden acaba en punto y coma.

Las ordenes anteriores son un ejemplo de mapa de Arduino. Se denomina mapeado (horrorosa palabra que proviene de la expresión inglesa mapping) a como se van asignando los pines como ENTRADA/SALIDA y como se activan.

Esto en s4a es muchísimo más simple y en la versión original solo se puede hacer para unos cuantos pines.
